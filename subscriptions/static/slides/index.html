04 — Dockerfile arxivdaily · 05 / 10

digest/Dockerfile

Patrón típico,
10 líneas.

FROM python:3.11-slim

WORKDIR /app

# 1. deps first → docker cache
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 2. then code
COPY . .

HEALTHCHECK --interval=5m --timeout=10s \\
  CMD python -m digest.healthcheck || exit 1

CMD ["python", "-m", "digest.main"]
01
python:3.11-slim 45 MB base — ni Alpine (issues con wheels) ni full Debian.
02
Requirements primero Cache de capas: si el código cambia, las deps no se reinstalan.
03
HEALTHCHECK declarativo Docker reinicia el container si el ping falla 3× seguidas.
04
Imagen final: 263 MB Push en ~14s a GHCR. Atomic pull en la VM.

07 — Lo que Docker resolvió arxivdaily · 08 / 10

Por qué este stack, y no otro

Ocho cosas que no tuve que reinventar.

01
Aislamiento
Un service que crashea no se lleva al resto. Process boundaries por defecto.
02
Reproducibilidad
Mi máquina = la VM. Mismo Python, mismas libs, mismo OS.
03
Memory limits
mem_limit: 120m en YAML. Sin systemd, sin cgroups a mano.
04
Restart automático
restart: unless-stopped — el container vuelve solo.
05
Volumes
Code = imagen. Data = volume. Borrás un container, los PDFs siguen.
06
Healthchecks
Una línea en el Dockerfile reemplaza un script de monitoring.
07
Atomic deploy
Push a GHCR · compose pull · up. Sin downtime visible.
08
Dev = prod
El mismo compose.yml corre en mi laptop y en la VM.

08 — Preguntas anticipadas arxivdaily · 09 / 10

Backup · responsabilidades del trade-off

Por qué dije que no a las cosas obvias.

¿Por qué no Kubernetes?
Una VM con 956 MB de RAM. El control plane de k8s ya pide ~2 GB. Para 5 servicios sin escalado horizontal, docker-compose es el sweet spot — k8s sería overengineering puro.
¿Y si Oracle revoca free tier?
Backups diarios cifrados a S3. El compose.yml es portable: misma imagen, otro proveedor. Hetzner o Fly.io cuestan ~$5/mes con downtime estimado de 10 minutos.
¿SQLite y no Postgres?
Single-writer, <10k filas/día, sin replicación necesaria. SQLite en WAL mode sobra. Postgres añadiría 200 MB de RAM y operación — sin beneficio medible al volumen actual.
¿Cómo manejás secrets?
.env excluido del git, chmod 600 en la VM. En CI vienen de GitHub Secrets. Hay 3 secrets (SMTP, sign key, arXiv token) — no justifica el costo de Vault o SOPS.
